Process Monitoring

A process is the instance of a program which is ebeing executed by the CPU. It contains the program code and its activity.

Important parameters are: PID (Process ID), priority, occupied memory, used CPU percentage

  • top → call the system manager to visualize all the processes

    • top -M → use the best measurment unit in the column of the allocated memory of each process

    • > P, > M → order processes for CPU percentage or memory percentage

    • > k + type PID + signal → kill the associated process. Signal can be 15 (sigterm) for close request, or 9 (sigkill) for forcing the closure

  • free -m → monitor memory status. -m display in megabytes

  • ps → display processes active in the current bash session

  • htop → like top but better look. It has to be installed

  • pgrep + name → display all the PIDs of the processes containing name

  • ps aux | grep + name → display all processes (and their specs) containing name

  • kill + PID → try to close the process with the sigterm signal.

    • kill -9 + PID → use sigkill instead

In Linux everything is a file. In the /proc/ directory, there is a directory for every process that is in execution (named as the PID of the process). Inside there are various other files and directories used by the OS to monitor the process. For example, in the cmdline file there is the bash command that started the process.
